🧾 Business Context
A web application that helps Ontario residents estimate their provincial and federal income tax, identify deductions/credits, 
and explore tax-saving strategies — built to demonstrate Azure 3-tier architecture.

Tier 1 — Presentation (Azure App Service / Static Web App)
#Requirement
BR-01 Users can input income details (employment, self-employment, investment, rental)
BR-02 Users can select applicable deductions (RRSP, childcare, union dues, etc.)
BR-03 Users can select applicable tax credits (Basic Personal Amount, Ontario Tax Reduction, etc.)
BR-04 The app displays a real-time tax breakdown (federal + Ontario provincial)
BR-05 Results show marginal vs. effective tax rate clearly
BR-06 Users can compare two tax scenarios side-by-side (e.g., with/without RRSP contribution)
BR-07 The app is responsive and accessible on desktop and Mobile browsers
BR-08Users can export/download their tax summary as a PDF or CSV

Tier 2 — Application Logic (Azure App Service — Backend API)
#Requirement
BR-09 The system applies current CRA federal tax brackets for calculation
BR-10 The system applies current Ontario provincial tax brackets and surtax rules
BR-11 The API validates all user inputs and returns meaningful error messages
BR-12 The system supports tax year selection (e.g., 2023, 2024, 2025)
BR-13 The API calculates CPP/EI contributions as part of the net income picture
BR-14 The system provides tax-saving recommendations based on user inputs (e.g., "Contribute $X more to RRSP to drop a tax bracket")
BR-15 All calculation logic is versioned so historic tax year rules remain accurate


Tier 3 — Data (Azure SQL Database / Cosmos DB)
#Requirement
BR-16 Tax bracket tables (federal + Ontario) are stored per tax year and are admin-updatable
BR-17 Deduction and credit reference data is stored and queryable by tax year
BR-18 Users can optionally create an account to save and retrieve past calculations
BR-19 All saved calculation sessions store inputs, outputs, and timestamp for audit
BR-20 Admin users can update tax rate tables without redeploying the application


Cross-Cutting / Non-Functional Requirements
#Requirement
BR-21 Authentication via Azure AD B2C for user accounts (optional/registered users)
BR-22 All data in transit encrypted via HTTPS (TLS 1.2+)
BR-23 PII (if stored) must comply with Canadian privacy standards (PIPEDA)
BR-24 App Service auto-scales based on load (to demo Azure scaling capability)
BR-25 Application and API logs shipped to Azure Monitor / Application Insights
BR-26 Infrastructure provisioned via Terraform or Bicep (IaC — great for your DevOps portfolio)

```

1. tax_years (id, year, is_active)

- 26% on $111,733 to $154,906
- 29% on $154,906 to $220,000
- 33% above $220,000
 
Ontario provincial brackets:
- 5.05% on first $51,446
- 9.15% on $51,446 to $102,894
- 11.16% on $102,894 to $150,000
- 12.16% on $150,000 to $220,000
- 13.16% above $220,000
 
Ontario surtax thresholds:
- 20% surtax on Ontario tax above $5,315
- 36% additional surtax on Ontario tax above $6,802
 
Common deductions: RRSP, TFSA note, Union Dues, Childcare Expenses, Moving Expenses
Common credits: Basic Personal Amount ($15,705 federal), Ontario Basic Personal ($11,865), Ontario Tax Reduction
 
Use 2024 rates for all three years as placeholder (note in comments).

```
In /backend/src/controllers/taxController.ts, implement:
 
POST /api/calculate
- Validate request body using Zod schema
- Call taxCalculationService.calculateTax()
- Return results as JSON
- Handle validation errors with 400 status
- Handle server errors with 500 status
 
GET /api/tax-data/:year
- Return all brackets, deductions, and credits for the given tax year
- Return 404 if year not found
 
Add input validation rules:
- tax_year: required, integer between 2020 and 2026
- employment_income: required, number >= 0
- rrsp_contribution: optional, number >= 0, max 29210
- All income fields: optional, default 0
```

## STEP 11 — Docker Compose for Local Dev
 
```

   - Image: mcr.microsoft.com/mssql/server:2022-latest
   - Port: 1433
   - Environment: SA password, accept EULA
   - Volume: mssql_data
 
2. backend service:
   - Build from /backend Dockerfile
   - Port: 5000
   - Depends on db
   - Environment variables: DB_SERVER, DB_NAME, DB_USER, DB_PASSWORD, PORT
 
3. frontend service:
   - Build from /frontend Dockerfile
   - Port: 3000
   - Depends on backend
   - Environment: REACT_APP_API_URL=http://localhost:5000
 
Also create:
- /backend/Dockerfile (Node 20 alpine, npm install, npm run build, npm start)
